description: 自动将刚才写好的代码提交并上传到 GitHub 仓库
---
当用户触发此指令时，请按照我们设定的《Git 自动化备份工作流》执行代码上传使命：

// turbo-all
1. 执行 `git status`，分析改动了哪些模块或文件。
2. 执行 `git add .` 将所有变更分批或全部放入暂存区。
3. 根据刚才分析出的主要改动，智能生成一个符合规范的中文提交信息。提交格式必须是：`git commit -m "[模块名] 简述变更内容"` （例如：`[MenuBar] 优化菜单图标显示`）。
4. 运行 `git push` 命令将代码推送到远端 GitHub。
5. 推送完成后，总结一下本次提交了什么内容，并用鼓励的中文告诉用户备份已经成功。
